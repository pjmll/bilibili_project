#  Bilibili 视频“破圈”预测数据集 

## 一、 数据集概述
本数据集专为探索和预测“B站视频破圈因子”而构建。经过严格的数据清洗与去重，最终包含 **58,869** 条极高纯度的视频特征数据。
* **核心任务：** 二分类预测（Binary Classification）。
* **最终产物：** `bilibili_week.txt` (无表头、Tab制表符分隔的 TSV 格式，可直接传至 HDFS / Spark)。

## 二、 数据来源与提取指南 (💡 队友必看)
数据集由多个独立数据源混合而成，正负样本比例健康。如果组员不需要全局数据，**只想使用其中某一个特定的子数据集进行实验，可以根据 `week_name` 或 `label` 字段进行快速过滤切分：**

### 🟢 正样本库 (爆款破圈视频，Label = 1)
代表被市场充分验证的高流量视频。
1. **《每周必看》栏目：** * **提取方法：** 筛选 `week_name` 包含 `"期"` 字样的数据（例：`2026第368期...`）。
   * **特点：** 最具含金量的官方周更榜单，2019-2026年全覆盖。
2. **外部开源扩充：** * **提取方法：** 筛选 `week_name` 为 `"飞桨外部数据集"` 的数据。
3. **入站必看**：api爬取，只有98条。
4. **热门排行榜**：api爬取，大约几百条。

### 🔴 负样本库 (普通未破圈视频，Label = 0)
代表 B 站庞大底层的普通内容生态，用于防止模型过拟合。
1. **各分区最新普通视频：**
   * **提取方法：** 筛选 `week_name` 为 `"普通最新视频"` 的数据。
   * **特点：** 从全站 70+ 细分子分区拉取，已剔除播放量为 0 的无效数据。

## 三、 数据字典 (15个核心特征)
以下是 `bilibili_week.txt` 中每一列数据对应的字段含义与顺序（严格对应 Spark/Pandas 的 Schema）：

| 列索引 | 字段名 | 数据类型 | 释义与作用 |
| :---: | :--- | :--- | :--- |
| 0 | `up_name` | String | **UP主昵称** |
| 1 | **`week_name`** | String | **所属期数/数据源标记** (⚠️ **提取子数据集的核心字段**) |
| 2 | `title` | String | **视频标题** (已清洗换行/回车符) |
| 3 | `desc` | String | **视频简介** (缺失值已用标题填充) |
| 4 | `view` | Integer | **播放量** (已清洗所有 view <= 0 的无效数据) |
| 5 | `danmaku` | Integer | **弹幕数** |
| 6 | `reply` | Integer | **评论数** |
| 7 | `favorite` | Integer | **收藏数** |
| 8 | `coin` | Integer | **投币数** |
| 9 | `share` | Integer | **分享数** (反映跨平台裂变能力) |
| 10 | `like` | Integer | **点赞数** |
| 11 | `rcmd_reason` | String | **官方推荐理由** (普通视频通常为空，已用标题填充) |
| 12 | `tname` | String | **视频分区** (如：单机游戏、搞笑等) |
| 13 | `his_rank` | Integer | **历史最高全站排名** (未上榜视频统一为 0) |
| 14 | **`label`** | Integer | **预测目标 (Target)**：`1` = 破圈爆款，`0` = 普通视频 |

## 四、 数据清洗规则声明
从原始采集的 73,639 条数据缩减至最终 58,869 条，执行了以下严格清洗：
1. **去重：** 根据 `UP主昵称 + 视频标题` 剔除重复抓取的冗余视频。
2. **剔除脏数据：** 强行过滤了 `view <= 0`、各项互动指标 `< 0` 以及 `dislike > 0` 的异常记录。
3. **降噪防错：** 清理了所有文本特征中的特殊制表符（`\n`, `\t`, `\r`），确保 Hadoop HDFS 读取时绝不发生“串列/错位”事故。
